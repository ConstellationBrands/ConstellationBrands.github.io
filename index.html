Welcome to Constellation Brands Public API! (Beta Version)
 

The goal of this API is to provide developers access to our wine product information in an open and easy-to-use manner.

The API provides the following information about our products:

  • Brand Name
  • Sub-Brand Name
  • Varietal
  • Appellation
  • Vintage
  • Taste profile tags
  • Assets

The API provides a common interface allowing other applications to access wine data. The framework is based on open-standard REST based web service technologies which allow for maximum flexibility and compatibility with other systems. The API functionality can only be used by authorized customers who have the valid API Key. Below are the list of functions that are exposed through the API.



API Calls  Endpoint  Description
Wines http://api.cbrands.com/beta/wines Retrieves the list of all active Brand Names along with their assets.


Authentication  

Query string parameter apiKey is used to authenticate API calls.

Making a request  

All API request use HTTP GET method so creating a request is simply a matter of adding apiKey parameter to endpoint URL. Assuming your API key is test_key the request will be

http://api.cbrands.com/beta/wines?apiKey=test_key

Getting JSON or XML response type  

The default response type is JSON. In order to get XML data you can add .xml to endpoint URL.

http://api.cbrands.com/beta/wines.xml?apiKey=test_key

Response codes and errors  

The API uses standard HTTP response codes.

200 OK - the request was processed successfully.

401 Unauthorized - unauthorized or invalid API key.

404 Not found - the server has not found anything matching the URI given.

429 Too Many Requests - too many requests in a given amount of time.

500 Internal Server Error - the server encountered an unexpected condition which prevented it from fulfilling the request.
